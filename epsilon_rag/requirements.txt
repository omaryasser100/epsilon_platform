# RapidOCR + onnxruntime-gpu — PaddleOCR's PP-OCRv3 weights exported to
# ONNX, run via onnxruntime on the same CUDA device as PyTorch.
#
# Why this stack over Surya / EasyOCR / paddleocr:
#   • CRNN+CTC architecture — has a "blank" output token, so it can
#     naturally produce empty text on non-text crops. Eliminates the
#     transformer-decoder hallucination loops Surya was prone to on
#     figure regions misidentified as text.
#   • Native per-line confidence scores → simple threshold filter
#     (settings.ocr_min_confidence) replaces a "is this gibberish"
#     classifier.
#   • Pure ONNX runtime → no paddlepaddle CUDA pinning matrix; coexists
#     cleanly with torch on the same GPU.
#
# Model files (Arabic PP-OCRv3) auto-download from SWHL/RapidOCR on
# Hugging Face into HF_HOME on first init — see pipeline/ocr.py.
#
# onnxruntime-gpu 1.19 supports CUDA 12.x — matches the torch CUDA 12.1
# wheels above. Installing onnxruntime-gpu uninstalls any plain
# onnxruntime that transitive deps pulled in (they share package
# namespace), which is what we want.
rapidocr-onnxruntime==1.3.24
onnxruntime-gpu==1.19.0
